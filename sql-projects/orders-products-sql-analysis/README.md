*# Orders \& Products SQL Analysis*



*## Objective*

*Analyze orders and products data to answer key business questions such as:*

*- Total revenue*

*- Top-selling products*

*- Customer purchase behavior*



*## Database*

*Tables used:*

*- products (product\_id, product\_name, category, price)*

*- orders (order\_id, product\_id, quantity, order\_date)*



*## Tools*

*- MySQL*

*- SQL (Joins, Aggregations, Group By, Subqueries)*



*## Business Questions Answered*

*- What is the total revenue generated?*

*- Which products generate the highest revenue?*

*- Which categories perform best?*

*- What are the monthly sales trends?*



*## Key Learnings*

*- Writing INNER JOIN between orders and products*

*- Using GROUP BY and aggregate functions*

*- Filtering with WHERE and HAVING*

*- Writing clean, readable SQL queries*



*## How to Run*

*1. Run `schema.sql` to create tables*  

*2. Run `data.sql` to insert sample data*  

*3. Run queries from `queries.sql` to get insights*
